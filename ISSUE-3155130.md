# ISSUE-3155130: "Edit summary" toggle accessibility issues

**Original issue:** https://www.drupal.org/project/drupal/issues/3155130

## Plain-language summary

- The "Edit summary" and "Hide summary" toggle buttons for text fields with summaries (like the Body field) have multiple accessibility and usability problems:
  - The toggle buttons are placed inside labels for other fields, which is invalid HTML and confuses screen readers.
  - The accessible name of the textarea changes when toggling, which is disorienting.
  - The "Edit summary" button disappears when activated, and a new "Hide summary" button appears, making the UI unpredictable.
  - The summary textarea appears before the button that reveals it, which is not the expected order for disclosure controls.
  - In multi-value fields, the toggle button can be invisible but still focusable, failing WCAG 2.4.7 (Focus Visible).
  - All summary fields are named the same, making it hard to distinguish them for users of assistive tech.

## Local URL(s) to test

- Add or edit content with a long text field that has a summary (e.g., Article > Body field):
  - [http://drupal-core.ddev.site/node/add/article](http://drupal-core.ddev.site/node/add/article)

## How to quickly assess

1. Log in as an admin (`ddev drush uli` for one-time login).
2. Go to a content type with a long text + summary field (e.g., Article > Body).
3. Tab to the "Edit summary" button and activate it.
4. Observe if the button disappears and a new button appears, and if the summary field appears before or after the button.
5. Check the accessible names of the textareas before and after toggling (using browser dev tools or a screen reader).
6. For multi-value fields, check if the toggle button is visible and operable for each value.
7. Assess if all summary fields are distinguishable by name.

## Patch status

- Patch available: `core-summary-a11y-3155130-6.patch` (may not apply cleanly or pass tests)
- The patch wraps the summary field in a `<details>` element to improve disclosure semantics, but may need further work.
