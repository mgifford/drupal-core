# ISSUE-3046089: Accessibility of Main toolbar items and internal editing

**Original issue:** https://www.drupal.org/project/drupal/issues/3046089

## Plain-language summary

- Some toolbar buttons in Drupal Core appear as a single control but are actually groups of controls when accessed with screen readers (JAWS) or keyboard navigation.
- Using Tab on the main toolbar skips important navigation items, making it hard for keyboard and assistive tech users to access all CMS features (e.g., adding/editing content).
- Contextual "open configuration options" buttons (like "Edit") lack clear context or ARIA labeling, so users may not know what they do when tabbing or using a screen reader.
- The issue affects both the main admin toolbar and contextual links/buttons throughout the admin UI.

## Local URL(s) to test

- Main admin toolbar: [http://drupal-core.ddev.site/admin](http://drupal-core.ddev.site/admin)
- Content admin: [http://drupal-core.ddev.site/admin/content](http://drupal-core.ddev.site/admin/content)
- Front page (contextual links): [http://drupal-core.ddev.site/](http://drupal-core.ddev.site/)

## How to quickly assess

1. Log in as an admin (`ddev drush uli` for one-time login).
2. Use only the keyboard (Tab/Shift+Tab) to move through the toolbar and contextual links.
3. Check if all top-level toolbar items are reachable and in logical order.
4. Expand/collapse any menu sections and verify state is announced (look for `aria-expanded`).
5. For contextual "open configuration options" buttons, check if their purpose is clear out of context (e.g., via screen reader or by inspecting ARIA labels).
6. Optionally, use a screen reader (VoiceOver, NVDA, JAWS) to confirm findings.
